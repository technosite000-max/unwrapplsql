What is Wrapped PL/SQL?

A complete guide to Oracle's PL/SQL wrap utility

Understanding Oracle PL/SQL Wrapping

Oracle PL/SQL wrapping (also called obfuscation) is a security feature that converts readable PL/SQL source code into an encoded format that cannot be easily understood by humans. When you "wrap" PL/SQL code, Oracle transforms it using the wrap utility or the DBMS_DDL.WRAP procedure.

Why Wrap PL/SQL Code?

  • Intellectual Property Protection: Prevent competitors from viewing proprietary business logic
  • Security Through Obscurity: Hide sensitive implementation details (passwords, encryption keys, table structures)
  • Deployment Requirements: Some third-party vendors require wrapped code for licensing compliance
  • Accidental Modification Prevention: Discourage unintended changes to critical code

How Oracle Wrap Works

The wrap utility performs a one-way encoding (not encryption) of your PL/SQL source:

  1. Tokenizes the PL/SQL source code
  2. Applies a Base64-like encoding algorithm
  3. Adds a header indicating "wrapped" status
  4. Outputs obfuscated code that Oracle can still compile

Can Wrapped Code Be Unwrapped?

The short answer: No, not completely. Oracle's wrap utility is designed as a one-way transformation.

Why Full Recovery Isn't Possible

  • The wrapping process discards original formatting and comments
  • Variable names are lost or replaced with generic identifiers
  • The encoding is lossy—some information is permanently removed
  • Oracle does not provide an official "unwrap" utility

What CAN Be Recovered?

While full source recovery isn't possible, some information can be extracted:

  • Package/Procedure/Function Names: Often visible in the header
  • String Literals: Hard-coded text values may survive
  • PL/SQL Keywords: Some may be recognizable in the encoded output
  • Structure Clues: Approximate size and complexity of the original code

Alternatives to Wrapping

If you need stronger protection, consider these alternatives:

  • PL/SQL Native Compilation: Compiles to machine code (harder to reverse)
  • Oracle Database Vault: Restricts access to source code views
  • Third-Party Obfuscation Tools: Commercial products with stronger algorithms
  • External Implementation: Move sensitive logic to compiled languages (Java, C)

Frequently Asked Questions

Is wrapped PL/SQL completely secure?

No. Wrapping is obfuscation, not encryption. Determined individuals with reverse-engineering skills can recover partial information. It's a deterrent, not absolute protection.

Can I recover lost source code from wrapped files?

Only partially. Always maintain backups of original source code. Wrapped code is not a substitute for version control.

Does wrapping affect performance?

No. Wrapped code compiles and executes identically to unwrapped code. The wrapping is only at the source level.

Can I wrap triggers and type bodies?

Yes. The wrap utility works on all PL/SQL program units including procedures, functions, packages, type bodies, and triggers.

Conclusion

Oracle's PL/SQL wrapping provides a basic level of source code protection. While not unbreakable, it serves as a practical deterrent against casual inspection. Use the UnwrapPLSQL tool to analyze wrapped code and understand what information might be exposed, but always maintain proper source control and consider additional security measures for truly sensitive logic.
